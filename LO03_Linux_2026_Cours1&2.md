🟦 第二部分：Shell 基础（6–17 页）

核心目标：理解 Shell 是什么

1️⃣ Shell 本质（第6页）
• Shell 是命令解释器
• 可以交互式执行
• 也是编程语言

2️⃣ Prompt 机制（8–9页）
• root 是 #
• 普通用户是 $
• PS1 可修改

3️⃣ Bash 能做什么（第10页）
• 历史命令
• 后台作业（jobs）
• alias
• 变量初始化

4️⃣ Shell 优缺点（11–12页）

这里其实在讲：

Shell 适合“胶水语言”，不适合复杂工程。

⸻

🟦 第三部分：文件系统 + 权限模型（18–35 页）

这部分非常重要，是 Linux 的核心思想。

⸻

🔹 文件系统结构（18–23页）

重点：
• 根目录 /
• /home
• /dev
• /proc
• /var
• 绝对路径 vs 相对路径
• . 和 ..
• ~

这是 Linux 的“空间结构模型”。

⸻

🔹 文件权限模型（25–33页）

重点：

-rwxr-x---

三组权限：

user | group | others

两种表示方式：
• 符号表示法：u,g,o + r,w,x
• 八进制表示法：chmod 751

这是 Linux 安全模型的基础。

⸻

🔹 硬链接 vs 软链接（34–35页）
• ln → 硬链接
• ln -s → 软链接

这部分在你未来理解 inode、文件系统结构时非常重要。

⸻

🟦 第四部分：IO 重定向 + 管道（36–42页）

这部分开始进入“Unix 精髓”。

重定向

> 覆盖输出
>
> > 追加输出
> > < 重定向输入
> > 2> 错误输出

/dev/null 是经典设计。

⸻

管道

ls -l | more

👉 Unix 思想：

小工具 + 管道 = 强大系统
