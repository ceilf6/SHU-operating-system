# Shell
Shell既是命令解释器，也是脚本语言

命令解释器：充当用户与操作系统之间的交互接口

解释器：用户输入命令，先经过语法检查、然后被运行

Linux中存在多种 Shell 解释器: 

- bash - Bourne again shell
- C shell
- Korn shell
- Z shell

所有 Shell 均可在同一 Linux 系统中共存，管理员会在用户配置文件

`/etc/passwd`

中，为每个用户设定默认 Shell

每个 Shell 都有独立的工作环境与编程语言

## 工作环境

支持以交互模式执行命令，也可以通过命令文件（脚本文件）执行命令

在交互模式下，bash 会在屏幕上显示一个命令提示符号（也称为 prompt/invite）对于管理员root用户，该提示符默认以 `#` 加一个空格结尾，其他普通用户默认以 `$` 加一个空格

命令提示符由三部分构成：用户登录名 + 主机名 + 当前目录的绝对路径

若希望缩短命令提示符串，那么只需要修改 Shell 预定义变量 **PS1** 即可

### bash Shell

- 调用历史记录，避免重复输入相同命令
    
    缓存
    
- 对当前输入的命令文本进行行内编辑
- 管理后台运行任务 jobs
- 初始化配置变量，包括编辑器的提示符字符串、默认搜索路径，创建命令别名（快捷命令 - 用 alias ）

## 优势

解释型语言，错误可以被定位和处理，无需对整个程序重新编辑

处理字符串，避免类型错误

适配应用的快速原型开发：管道 tubes 、命令替换与变量替换机制，支持通过组合 Unix 环境中有的命令构建应用

可以连接不同编程语言编写的组件

## 缺点

语法严格，一个空格字符就可能导致语法错误

实现同一功能可以用多种语法

部分特殊字符（如括号）在不同上下文下含义不同：括号可用于表示命令列表、函数定义，或规定算术表达式的运算优先级

# 命令行
## 语法

```tsx
命令名 [选项] [参数]
```

命令行由一个或多个空白字符分隔的若干「单词」组成。这里的「空白字符（blanc）」指水平制表符（tab）或空格字符

- 选项（option）通常以单个连字符开头（例如：-a），或按语法规则以两个连续的连字符开头（例如：--version）。
- 参数（arguments）指命令需要执行操作的目标对象。

多个命令用 ; 进行分隔
